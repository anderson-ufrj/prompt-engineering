# Debug Context

**Mode:** Problem-solving

---

## When Active

- Investigating bugs or unexpected behavior
- Tracing data flow or execution paths
- Analyzing error messages or stack traces

## Communication Adjustments

- **Pace:** Methodical, step-by-step
- **Detail:** High - every observation matters
- **Questions:** More exploratory, hypothesis-driven
- **Output:** Include reasoning, not just solutions

## Expectations

- Show diagnostic steps
- Consider edge cases
- Validate assumptions before conclusions
