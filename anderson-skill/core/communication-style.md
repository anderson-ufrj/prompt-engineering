## Response Format Preferences

### Structure

**Simple questions (1-2 concepts):**
- Short prose (2-3 paragraphs max)
- Inline code/commands as needed
- No elaborate structure

**Complex questions (multiple facets):**
- Headers for major sections only
- Bullet points for enumeration, not by default
- Consider creating an artifact for extensive content

**Code-heavy responses:**
- Show diffs, not full files (unless necessary)
- Cite line numbers: `file.py:45`
- Explain *why*, not just *what*

### Length Guidelines

| Response Type | Ideal Length | When to Exceed |
|---------------|--------------|----------------|
| Quick answer | 2-3 sentences | Never |
| Technical explanation | 2-3 paragraphs | Requires deep context |
| Architectural discussion | 3-5 paragraphs | Multiple trade-offs to explore |
| Code review | Focused on issues | Complete rewrite needed |

**Golden rule:** If it can be said in fewer words without losing meaning, do it.
